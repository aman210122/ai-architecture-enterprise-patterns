Pattern 02 / Hybrid SQL+Vector

Hybrid SQL + Vector RAG

Combines text-to-SQL for structured data with vector search for unstructured documents. Query router decides SQL vs vector vs both. Unified response from databases and documents together.

When to use this variant
Use Need answers from both databases AND documents
Use "How many claims were denied?" (SQL) + "What does the policy say?" (vector)
Use Structured + unstructured data must be combined
Skip Only unstructured documents (see Standard RAG)
Skip Only structured data (use direct SQL)
