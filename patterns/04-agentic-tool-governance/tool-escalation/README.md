# Variant: Tool Permission Escalation

**Dynamic permission elevation at runtime. Agent starts read-only, requests elevated access through approval flow.**

## Live Demo
**[Launch](https://aman210122.github.io/ai-architecture-enterprise-patterns/patterns/04-agentic-tool-governance/tool-escalation/index.html)**

Scenarios: Read-only call, escalation denied, escalation approved, read-then-write sequence, escalation timeout.

**Use:** Agent needs to read first then write. Risk-based permission elevation. Gradual trust during execution. Principle of least privilege with dynamic elevation.
